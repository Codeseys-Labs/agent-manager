# Releasing agent-manager (`am`)

Distribution: **GitHub Releases** (5 platform binaries + `am-acp-shell` +
`checksums.sha256`, consumed by [`install.sh`](install.sh)) and **Homebrew**
(`Formula/am.rb`, regenerated by CI). **npm is deferred** (the unscoped name is
taken â€” see the CHANGELOG note); do not add an npm publish step until that
changes.

A tag `vX.Y.Z[-rc.N]` pushed to the repo triggers
[`.github/workflows/release.yml`](.github/workflows/release.yml). Everything
below the tag is automated; your job is the pre-flight and the CHANGELOG.

## Versioning

- **Stable:** `vX.Y.Z`. **Prerelease:** `vX.Y.Z-rc.N` â€” **dotted** (`-rc.8`, NOT
  `-rc8`). The dot makes `N` a numeric SemVer identifier, so `rc.8 < rc.10`. The
  no-dot form is a single ASCII identifier and sorts `rc.10 < rc.7`, which
  breaks every semver-aware tool. `scripts/bump-version.sh` and the CI
  `assert-version` job both reject the no-dot form.
- The build **bakes the version from the git tag** (`GITHUB_REF_NAME`) via
  `--define`, so the **tag is the source of truth**. `package.json` must match
  it â€” CI asserts this before building.

## Pre-flight (on an up-to-date `main`)

1. `git switch main && git pull --ff-only`
2. `bun install --frozen-lockfile && bun test && bun run typecheck && bun run lint`
   â€” all green, or stop.
3. **Curate `CHANGELOG.md`:** ensure everything since the last tag is under
   `## [Unreleased]` in the right Added/Changed/Fixed/â€¦ buckets. Do **NOT** add
   the date or version heading â€” CI stamps those. The bullets you leave here
   ship **verbatim** as the GitHub Release notes.
4. **Smoke-build locally** so you catch a version-threading or Silvery-patch
   break before tagging:
   ```sh
   VERSION=X.Y.Z-rc.N bun run build -- --target bun-linux-x64
   ./dist/am-linux-x64 version        # must print X.Y.Z-rc.N (not 0.0.0-dev)
   ```

## Cut the release

5. Run the driver:
   ```sh
   ./scripts/bump-version.sh X.Y.Z-rc.N --dry-run   # preview
   ./scripts/bump-version.sh X.Y.Z-rc.N             # for real
   ```
   It refuses a dirty tree / already-used tag, writes `package.json`, commits
   `chore: bump version to â€¦`, annotates tag `vX.Y.Z-rc.N`, and pushes the
   commit + tag.

   **Safer variant (recommended):** push the bump commit first, wait for `main`
   CI to go green, *then* push the tag â€” so a broken build never produces a
   tagged release:
   ```sh
   git push origin HEAD          # bump commit only
   # â€¦watch main CI go greenâ€¦
   git push origin vX.Y.Z-rc.N   # now trigger the release workflow
   ```

## What CI does (do not do by hand)

- **`assert-version`** (gates everything): tag == `package.json` version,
  SemVer/dotted-prerelease shape, `[Unreleased]` non-empty.
- Builds 5 platform binaries + `am-acp-shell`, ad-hoc-signs the darwin binaries,
  asserts the linux binary self-reports the tag, generates `checksums.sha256`.
- Extracts `[Unreleased]` as the release notes and creates the GitHub Release
  (auto `--prerelease` for `-rc`/`-alpha`/`-beta`, so it is **not** marked
  "Latest").
- Regenerates `Formula/am.rb` and **stamps `CHANGELOG.md`** (`[Unreleased]` â†’
  `[X.Y.Z] - DATE`, opens a fresh `[Unreleased]`), committing both back to
  `main`.

## Verify (after the run is green)

6. On a clean machine (or `--dry-run`):
   ```sh
   curl -fsSL https://raw.githubusercontent.com/Codeseys-Labs/agent-manager/main/install.sh | sh
   am version    # == X.Y.Z-rc.N
   ```
7. Confirm the Release page lists all binaries + `checksums.sha256`, and that
   `--prerelease` is set for rc tags (the "Latest" badge is unchanged).
8. Confirm the CI bot commit landed on `main` (`package.json`, dated
   `CHANGELOG`, updated Formula).

## If it goes wrong (recovery)

- Delete the GitHub Release **and** the tag:
  ```sh
  gh release delete vX.Y.Z-rc.N --cleanup-tag
  # or: git push --delete origin vX.Y.Z-rc.N
  ```
- If CI already pushed the stamp/Formula commit to `main`, revert it:
  ```sh
  git revert <sha> && git push
  ```
- Fix the cause, **re-bump to the NEXT rc** (never reuse a tag), re-push.
